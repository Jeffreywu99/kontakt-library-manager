# Changelog

## [0.3.0] - 2026-05-21

### Changed
- **核心架构重构**："入库即显示，删除即消失"
  - 主列表只显示已注册的库（有注册表/XML/JSON 记录）
  - 删除类型概念（标准库/非标准库/注册库）
  - 删除隐藏功能
  - 入库时自动把父目录加入库文件夹列表
- **UI 简化**：
  - 左侧面板：全部 / 用户分类 / 未分类
  - 表格删除"类型"列
  - 文件夹管理对话框简化为单一列表
- **数据模型简化**：
  - `LibraryEntry` 删除 `library_type` 和 `hidden` 字段
  - `data.json` 删除 `custom_libraries`、`hidden_libraries`、`show_registry_libraries`

### Fixed
- **入库"可见但不可用"问题**：使用 .nicnt 中的 `RegKey` 作为注册表键名，而非文件夹名
  - 之前：文件夹名 `Tina Guo vol 2` → 注册表键名 `Tina Guo vol 2`（错误）
  - 现在：.nicnt `RegKey` = `Artist Series - Tina Guo vol 2` → 注册表键名正确

## [0.2.1] - 2026-05-20

### Fixed
- 入库/删除库完全失效：恢复 v0.2.0 误删的注册表/XML/JSON 扫描逻辑
- 注册表写入容错：3 个路径全部失败才报错，而非任意一个失败就中断
- 注册表删除：支持递归删除含子键的注册表项
- 查看来源位置按钮 NameError：补回误删的 `import subprocess`

## [0.2.0] - 2026-05-20

### Added
- 刷新按钮：重新扫描所有库文件夹
- 文件夹大小显示（MB / GB 自动切换）
- UI 防抖优化：快速点击不再卡顿，备注输入延迟保存

### Changed
- 大幅简化：移除注册库显示、清理残留、打开注册表等冗余功能
- 只显示用户添加的文件夹中的库（标准 + 非标准）
- 左侧面板简化为：全部 / 标准库 / 非标准库 / 用户分类 / 未分类 / 隐藏

### Fixed
- 注册表操作覆盖 HKCU 路径
- 批量入库对话框
- 备注保存性能优化

## [0.1.0] - 2026-05-19

首个可用版本。
