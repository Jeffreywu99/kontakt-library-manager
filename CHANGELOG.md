# Changelog

## [0.5.0] - 2026-05-21

### Added
- **首次启动自动发现**：首次打开软件自动扫描注册表，显示所有已注册的 Kontakt 音色库（Visibility=3 + HU + JDX），无需手动配置库文件夹
- **增量扫描**：后续启动通过缓存对比注册表变化，毫秒级完成
- **强制完整扫描**：刷新按钮右键菜单支持清除缓存并完整重建
- **扫描缓存机制**：`data.json` 新增 `scan_cache` 持久化发现结果

### Changed
- **扫描逻辑重构**：从文件夹驱动改为注册表驱动
  - `list_kontakt_libraries()` — 遍历三个注册表位置，三要素过滤，ContentDir 去重
  - `auto_discover_all()` — 首次全量发现 + 自动提取库文件夹
  - `scan_incremental()` — 增量对比缓存，检测新增/移除
- **显示名称**：统一从 `.nicnt` 的 `<Name>` 字段读取，与 Kontakt 内显示一致
- **状态栏**：显示库数量和上次扫描时间

### Fixed
- 不再误显示 Battery 4 Factory Library、Massive X Factory Library 等非 Kontakt 音色库
- 自动排除路径含 `\Third Party\Native Instruments\presets` (Arturia)、`\Universal Audio\Plug-Ins\` (UAD)、`\NKS\` 等非 Kontakt 条目
- ContentDir 去重解决同一库有多个注册表键名的问题

## [0.4.1] - 2026-05-21

### Fixed
- **批量入库勾选框点击偏移**：修复点击区域不在勾选框位置的问题
- **批量入库误入库保护**：已入库的库默认不勾选，放在列表底部，有分隔线标识

### Changed
- **添加标准库目录交互优化**：
  - 添加目录时自动扫描未入库的库
  - 发现未入库库时询问是否全部入库
- **移除标准库目录交互优化**：
  - 移除前显示该目录中已入库库的数量
  - 询问是否同时从 Kontakt 中移除
- **非标准库目录交互优化**：
  - 移除时显示确认对话框，说明只影响软件显示

### Added
- **兼容性说明**：在 README 中明确支持的 Kontakt 版本和用户群体

## [0.4.0] - 2026-05-21

### Added
- **非标准库目录**：在"管理库文件夹"中可添加非标准库目录
  - 自动扫描目录下的子文件夹作为非标准库
  - 无需逐个添加，一键扫描
- **音色库搜索**：工具栏新增搜索框，实时过滤音色库
- **启动画面**：应用启动时显示加载画面

### Changed
- **扫描逻辑重构**：完全基于用户添加的目录显示内容
  - 标准库目录：只显示有 .nicnt 且已注册的库
  - 非标准库目录：自动扫描所有子文件夹
  - 不在用户目录中的库不再显示（彻底过滤 SWAM、UAD 等）
- **左侧分类面板**：新增"标准库"和"非标准库"两个默认分类
  - 非标准库独立管理，不参与注册表逻辑
  - 非标准库不显示 nicnt 警告
- **UI 重设计**：VS Code 风格暗色主题
  - 更高级的灰色调配色
  - "存在"列改为淡色圆点（●/○）
  - 管理员状态改为淡灰色小字
  - 扁平化按钮样式
- **管理库文件夹对话框**：分为标准库目录和非标准库目录两个列表
- **批量入库对话框优化**：
  - 加宽复选框列，修复重叠问题
  - 增大对话框尺寸

### Removed
- 冗余的旧别名方法（`list_roots`/`list_folders` 等）
- 复杂的注册表过滤逻辑（`_is_kontakt_library`）
- 工具栏的"添加非标准库"按钮（集成到管理库文件夹）
- "未分类"分类项（改用标准库/非标准库分类）

## [0.3.0] - 2026-05-21

### Changed
- **核心架构重构**："入库即显示，删除即消失"
  - 主列表只显示已注册的库（有注册表/XML/JSON 记录）
  - 删除类型概念（标准库/非标准库/注册库）
  - 删除隐藏功能
  - 入库时自动把父目录加入库文件夹列表
- **UI 简化**：
  - 左侧面板：全部 / 用户分类 / 未分类
  - 表格删除"类型"列
  - 文件夹管理对话框简化为单一列表
- **数据模型简化**：
  - `LibraryEntry` 删除 `library_type` 和 `hidden` 字段
  - `data.json` 删除 `custom_libraries`、`hidden_libraries`、`show_registry_libraries`

### Fixed
- **入库"可见但不可用"问题**：使用 .nicnt 中的 `RegKey` 作为注册表键名，而非文件夹名
  - 之前：文件夹名 `Tina Guo vol 2` → 注册表键名 `Tina Guo vol 2`（错误）
  - 现在：.nicnt `RegKey` = `Artist Series - Tina Guo vol 2` → 注册表键名正确
